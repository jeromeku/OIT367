Project Title: Exploring the Correlational Structure of the Stock Market**

Abstract

The goal of my project was to use unsupervised machine learning and other statistical measures to dissect the volatility of stocks and other asset classes. The purpose was not so much prediction or creating trading models, though they could serve as means for idea generation, as it is diagnostic and understanding how the overall correlational structure of the markets has evolved over the past two decades.

Note that I pivoted twice in project idea:

  • I originally had sourced a project with a local start-up (per my original proposal). This data ended up being a bit too messy (completely in Chinese) and too much of my time was spent trying to massage the data into workable form.

  • My next idea was to work with Yodlee data but the iteration time between data requests from the query team ended up taking too long.

Preliminaries

Tools

I did all analysis in Python. I used iPython Notebook, an interactive "lab" notebook to do the analysis and write-up.

Libraries used include:

  • Pandas: provides tools for data-munging and analysis. Especially well-suited for time-series data.
  • scikit-learn: machine learning algorithms

Data

I used downloaded and pieced together several datasets from various sources:

  • Wharton Research Data Services (WRDS): I used the CRSP and COMPUSTAT databases to download metadata as well as pricing data for the S&P 500. Specific services used were Index Constituents, which listed the years that certain stocks listed as part of the S&P 500. Using this along with the Securities Daily file, I was able to create a time series matrix that at any time-point contained only the 500 stocks in the S&P at that time. See code for details on this process.

  • DataStream: This service provides data on international equities, equity indices, bond indices, real estate indices, and commodities. This service is available in Jackson Library in the Makena Trader Pit. Specific datasets downloaded are the S&P Sector Indices.

  • Quandl: This is online data aggregation service specifically for the capital markets and provides an API for accessing datasets. See Quandl for further details. I used this to download the daily VIX series.

Exploring Correlation of the S&P 500

For my initial analysis, I wanted to explore the correlational structure of the S&P 500 over the past decades and see how this changed over financial crises and other market phenomena. In order to do this, I first needed to gather a list of S&P constituent companies over the past 20 years in addition to their daily closing prices. Since this list is dynamic (stocks are added / removed from the index every year), I had to massage together a few datasets to get the data in required form.

This was done is a few steps:

  • Download S&P 500 index constituents from WRDS CRSP/Compustat Database. The data was structured as a Stock x Feature matrix. Important features included the start and end dates on the S&P 500.
  • Download daily closing prices of all companies included in the index at some point in the past 20 years. This was done using the Securities Daily Compustat database in WRDS.

From the daily security prices, I was able to construct a time series dataframe. Using the start / end dates from the S&P constituents download, I created an indicator matrix of the same dimensions as the daily security prices where an entry is 1 if the stock was on the S&P 500 for that given day and 0 if not. Using this second matrix as a mask, I multiplied the time series prices by this mask to create a new time series dataframe where each day (row in the matrix) now only had prices for stocks that were in the S&P 500 at that time.

Additionally, I mapped each stock to its respective GICS sector (e.g., Consumer Discretionary, Energy, IT, etc.) in order to aid exploration of the data.

To characterize correlation over the past 20 years, we wrote a function that calculates a rolling PCA (see utils.py):

  • Calculate the correlation matrix for the 500 stocks on the S&P over a given time span (here, we use 252 days, which is the number of trading days in a year)
  • Calculate PCA for the correlation matrix
  • Roll the period forward by a given increment (we use 30 days here) and repeat the calculation
  • Continue until we reach the desired end date

What this allows us to do is visualize how the correlational structure is changing over a given span of time. More specifically, we can plot the number of eigenvalues / eigenvectors needed to explain a given level of variance and see how this changes over time:

Dispersion

Next we explore another dimension of volatility known as dispersion. Whereas correlation measure the co-movement of assets in a portfolio, dispersion measures the degree of variation of the returns in a portfolios' constituents.

Dispersion is defined as:

inwi(riP)2
where wi and ri are the weight and return of the ith asset and P is the portfolio return.

We explore this measure using the S&P 500 as the reference portfolio and the 500 constituent stocks as the components. The weights in this case are the market capitalizations of the stocks.

See S&P Indices whitepaper Dispersion: Measuring Market Opportunity by Tim Edwards and Craig Lazzara for further details.

What we generally observe over the past decade is that the two series have tended to move in opposite directions. From 2005-8, the S&P 500 was rising as one -- that is there was low variation in the individual component returns during this time.

Subsequently during the 2008-10 financial crisis, the S&P 500 fell, disproportionately affect certain stocks, as dispersion rose dramatically during this period.

From 2010 to now, we see that there has been a bull-market, with relatively low dispersion between individual stocks, indicating that you would probably be better off holding the market portfolio (i.e., an index fund) than you would trying to pick individual stocks.

def remove_border(axes=None, top=False, right=False, left=True, bottom=True):
    """
    Minimize chartjunk by stripping out unnecesasry plot borders and axis ticks
    
    The top/right/left/bottom keywords toggle whether the corresponding plot border is drawn
    """
    ax = axes or plt.gca()
    ax.spines['top'].set_visible(top)
    ax.spines['right'].set_visible(right)
    ax.spines['left'].set_visible(left)
    ax.spines['bottom'].set_visible(bottom)
    
    #turn off all ticks
    ax.yaxis.set_ticks_position('none')
    ax.xaxis.set_ticks_position('none')
    
    #now re-enable visibles
    if top:
        ax.xaxis.tick_top()
    if bottom:
        ax.xaxis.tick_bottom()
    if left:
        ax.yaxis.tick_left()
    if right:
        ax.yaxis.tick_right()

def getRecessionBands(start_date='1965',data_file='data/FRED/USREC.csv'):
    '''Returns list of 2-tuples of start, end dates for recessions per NBER
        Arguments:
            start_date: Date from when to start retrieving recession bands
            data_file: filepath for FRED recession band file
        Returns:
            List of tuples corresponding to the recession start/end spans since
            the requested start date
    '''
    recessions = pd.read_csv(data_file,parse_dates=['Date'],index_col=0)
    recessions = recessions[start_date:]
    recession_changes = recessions.diff().dropna()
    recession_dates = recession_changes[recession_changes.Value != 0].index
    assert len(recession_dates) % 2 == 0
    
    recession_bands = [(recession_dates[i],recession_dates[i+1]) for i in range(0,len(recession_dates),2)]
    return recession_bands

def formatSecondaryLegend(left_ax,right_ax):
    '''Formats the legend for plots with dual axes
        Arguments:
            left_ax: axis object corresponding to left axis
            right_ax: similar for right axis
        Returns:
            tuple with 2 entries, one corresponding to the lines argument
            for a call to plt.legend and the second the labels argument
    '''
    left_lines,left_labels = left_ax.get_legend_handles_labels()
    right_lines, right_labels = right_ax.get_legend_handles_labels()
    left_labels[0] += " (L-axis)"    
    right_labels[0] += " (R-axis)"
    
    left_lines.extend(right_lines)
    left_labels.extend(right_labels)
    
    return (left_lines,left_labels)
